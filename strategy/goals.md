# Goals

Current, concrete objectives beneath the vision (`VIS-001`). Owned by the user; agents read these to judge priority (triage, picking work) but never edit them. Revisit roughly monthly.

## Now (July 2026)

1. **Ways of working established** — knowledge system live; development, product, and QA practices researched and documented (EPIC-007), so agent loops run reliably with quality gates.
2. **Foundation built** — EPIC-001 done: theory core (incl. scales/arpeggios), fretboard + chord-diagram rendering, app shell, typed persistence.
3. **Guided practice loop, end to end** — one guitarist (the owner) can open the app, see today's plan on the dashboard, run a guided lesson (scales or arpeggios, self-graded), and find the session in history. The thinnest vertical slice of EPIC-008 + EPIC-011 + EPIC-012.
4. **De-risk the hard parts** — research done for notation/tab rendering (EPIC-009) and recording/scoring feasibility (EPIC-010) before committing UI work to either.

## Later (unordered)

- Notation + tablature in the practice view (EPIC-009, after research)
- Recording + scoring of takes (EPIC-010, after feasibility spike)
- Voicing drills (EPIC-002), ii–V–I trainer through all twelve keys (EPIC-003)
- Metronome + structured sessions (EPIC-004)
- Repertoire tracker feeding the planner (EPIC-005)
- Deployed publicly as a static site
