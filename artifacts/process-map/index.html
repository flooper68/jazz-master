Routing table

Start with the trigger, then choose the process.

Each process has a narrow job. This keeps raw feedback, backlog decisions, implementation, and shipping from blending together.

Default path

Most product work follows this path.

1

Intake

Owner notes, QA findings, feedback, or observations become durable files.

2

Triage

Insights and issues are accepted, rejected, deferred, confirmed, or turned into tasks.

3

Prioritize

Actionable tasks and issues are ranked by blockers, owner direction, practice value, evidence, and size.

4

Dev loop

Pick, claim, plan, implement, review, test, record, ship, reflect.

5

QA review

The running app is inspected, and new issues or insights feed the next cycle.

Quality gates

Nothing ships just because it was implemented.

  • Independent code review before ship.
  • `bun run check` is the required gate before push.
  • Item verification steps must be executed literally.
  • Security review is required for sensitive surfaces.
  • Code and tracker updates ship in the same commit.

New convention

Markdown is source of truth. Artifacts are rendered companions.

Use Markdown for

Agent instructions, processes, work items, research, ADRs, and architecture maps.

Use artifacts for

Interactive visuals, decks, reports, screenshots, exports, and human walkthroughs.

`artifacts/<slug>/` with a README, source links, opening steps, and maintenance note.
