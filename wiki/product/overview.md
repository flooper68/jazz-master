# How the product works

Jazz Master is a web app that helps guitarists get better at jazz — a structured practice companion, not a chord-chart reference. The core promise (VIS-001) is **zero-tension practice**: open the app, pick up the guitar, and play. The app has already decided today's session from your level, goals, and history, and guides you through it exercise by exercise. It targets intermediate guitarists learning the jazz language — beginners in jazz, not on the instrument.

## The practice flow

The intended end-to-end session, once the guided-practice slice lands (strategy/goals.md "Now" item 3):

1. **Dashboard** shows today's plan, generated by the adaptive planner from a local profile plus practice history (EPIC-011, EPIC-012).
2. **Guided lesson** runs the plan exercise by exercise — drills like "play this ii–V–I in Bb, now in Eb, now with drop 2 voicings" — rendered guitar-first: fretboards, grip diagrams, and — for scale/arpeggio exercises since 2026-07-07 (TASK-038) — staff notation with tablature via VexFlow 5 behind a project-owned `<Notation>` component, opted in per exercise through a `'notation'` display hint (EPIC-008, EPIC-009; ADR-010). Since TASK-048, that score defaults larger, reserves its layout space, lets the player choose staff, TAB, or both with a persisted local preference, and can expand into an in-run focus mode with grading controls. Since TASK-047, each exercise panel can also play the exercise notes with sampled guitar timbre, loop them with click/count-in, and remember a slower practice tempo per exercise.
3. **Grading**: self-graded first; recording with in-browser pitch/timing scoring — VIS-001's riskiest bet — is researched **staged-go** (RES-014): monophonic offline-after-take analysis for v1, chord scoring staged later. The recommended real-guitar spike was skipped by accepted owner risk (NOTE-010), so EPIC-010 proceeds from RES-014 defaults and synthesized-test evidence. The runner now has the capture UI (mic permission, input meter, count-in, in-session replay) and, since TASK-043, decodes a stopped take locally, shows a 0-100 score with pitch/timing/completeness and per-note verdicts, and persists score metadata with the graded exercise while leaving raw audio transient. NOTE-012 moves human-only browser/device mic coverage to QA/product review.
4. **History**: the session lands in practice history; the dashboard shows streaks, scores, keys mastered — momentum (EPIC-012).

Content exists to feed drills and lessons, not as browsable reference — "the app decides, you play" is the differentiator.

## Module pillars

Each pillar maps to an epic (VIS-001): foundation (EPIC-001), chord voicings (002), progression trainer (003), practice tools/metronome (004), repertoire (005), ear training (006), curriculum & lessons (008), notation & tabs (009), recording & scoring (010), adaptive planner (011), dashboard & history (012), plus the multi-app platform track (013). Ways-of-working (007) is process, not product. Non-goals for now: social features, native mobile, notation *editing*, accounts/backends, bundled copyrighted material.

## How it works under the hood

Everything runs client-side; there is no backend and no account (ADR-002 — "logging in" means a local profile). Domain logic — notes, intervals, chord spelling, scales, arpeggios, fretboard positions, with enharmonics correct by construction — lives in the pure `@jazz-master/theory` package; curriculum content (exercises and lessons in `apps/web/src/content/`) references theory constructs rather than hard-coding notes, so a lesson is "the Bb major scale in position II", not a note list; pages compose reusable components (Fretboard, ChordDiagram) which render what theory computes; all persistence goes through typed localStorage stores in `apps/web/src/storage/` (details: architecture/overview.md), including per-exercise play-along tempos, notation preferences, scoring tolerance preferences, and score-only session metadata. Since ISSUE-005, `/app/profile` includes JSON backup export/import for those typed stores so local-first data has a user-owned recovery path for Safari/WebKit eviction, storage clearing, or browser moves. The shell is now hybrid (EPIC-013 / ADR-006, first slice shipped as TASK-021): Astro serves a barebones landing page at `/` and hosts the whole React practice app as a client-only SPA island under `/app/*` — so every product route above now carries the `/app` prefix (`/app/practice`, `/app/history`, …). The build targets Cloudflare Workers; a typed tRPC API surface now exists (TASK-023 — scaffolding only: a `/trpc/health` endpoint rendered as a small status chip in dev builds only since TASK-024; no product feature uses the server yet), and deployment is CI-only per ADR-009 — Cloudflare Workers Builds auto-deploys the dev environment at https://jazz-master.premysl-ciompa.workers.dev on every push to `main` (live since 2026-07-07, TASK-024), no deploy credential exists anywhere agents can reach, and there is no production environment — deliberately: the dev URL is the product's home, and the parked production task was abandoned outright (TASK-036, owner decision 2026-07-07, NOTE-008). The local-first product model is explicitly kept: practice state stays in localStorage; no accounts.

## Built today vs pending (as of 2026-07-08)

Done: app shell with routing and sidebar, the theory core, fretboard and chord-diagram components, and the persistence layer — EPIC-001 (foundation) is complete. EPIC-008 (curriculum & lessons) is complete: the exercise/lesson content model (TASK-011), the first lesson pack (TASK-012 — 10 scales/arpeggios lessons across 3 levels), and the guided practice runner (TASK-013) — from the Practice page a user starts a lesson, is guided exercise by exercise (fretboard positions, countdown, got-it/shaky/missed self-grades), and the session — completed or abandoned — persists as a `PracticeSession` record in the `sessions` store. EPIC-011 (adaptive planner) is complete: first-run onboarding or skip writes a local `PracticeProfile`, `/profile` edits it, and `/practice` shows a persisted Today's plan generated from profile, lesson metadata, session history, and date; each plan item has a reason, starts the runner, and ticks done from completed sessions. EPIC-012 (dashboard & history) is complete: the practice-history page (TASK-018) makes every persisted session findable at `/history` — grouped by day newest-first, filterable by area and time range, with grade tallies, incomplete markers, session-level machine scores, and drill-down to per-exercise self-grades/scores — and the dashboard v1 (TASK-019) replaces the `/` stub with the product's front door: today's plan with reasons and a one-click Start into the runner, current streak, minutes-this-week against the profile budget, and per-area status with needs-attention callouts (trend-over-time visuals wait for broader score history). EPIC-009 (notation & tabs) is complete as of 2026-07-07: every lesson-pack scale/arpeggio exercise renders an aligned staff + TAB pair below the fretboard in the runner (TASK-037 component, TASK-038 integration), VexFlow lazy-loads only when a notation exercise appears, and the bundle trim (TASK-039) cut that lazy chunk from 692 to 389 KB gzip via the single-music-font `vexflow/bravura` entry. TASK-048 added the post-epic readability layer: larger default score scaling, reserved score space, keyboard-scrollable overflow, staff/TAB/both controls, a persisted notation display preference, focus mode, and a smoke assertion that real SVG score content rendered. EPIC-014 (play-along) is complete: the runner lazy-loads sampled guitar playback only when Play is pressed, loops the exercise with click/count-in, lets the player slow the tempo up to the authored target BPM, and remembers that tempo per exercise. EPIC-010's monophonic v1 loop is implemented through TASK-043: the runner can request mic access, show a level meter, count in, record, replay, locally analyze the stopped take, show per-note score feedback, persist score metadata, and discard raw audio. NOTE-012 moved human-only desktop/mobile browser mic coverage to QA/product review rather than task completion gates; real-guitar plausibility remains something QA/dogfooding must watch. Everything user-facing beyond that — moving drills and chord close-enough scoring — remains pending: the other practice modules are stub pages. The guided-practice vertical slice (onboard → plan → practice → history → dashboard) is finished, and the notation/audio-scoring de-risking research (strategy/goals.md goal 4) is done as of 2026-07-07: notation rendering is decided and shipped (VexFlow 5, ADR-010/RES-013) and recording/scoring is staged-go (RES-014). The EPIC-013 platform migration's first-slice target is fully live: the app runs on Cloudflare Workers at https://jazz-master.premysl-ciompa.workers.dev, auto-deployed on every push to `main` (TASK-024 done 2026-07-07); only the gated database and production tasks remain.
